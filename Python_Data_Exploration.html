There are some variables that are worth noticing

index #6, #7, #10, #12: Can be left as is or categorize
index #0, #1, #2, #5, #11, #21, #22, #24 - #28, #30, #34 - #40: Are continuous variables
index #4, #29: Values in percentages, the type should be float
index #13, #23, #41, #43: Are Months and Years
index #3 - term: The column is object while it can be integer, further categorical
index #8 - emp-title: Might be useful, might be not, just unique strings
index #9 - emp-length: Description indicated integer categorical data, while actual said object
index #13 - issue_d: can be separated into Month and Year columns
index #14 - loan_status: certainly can be categorized but seems not standardized, need further inspection
index #15, #32, #33: only 2 unique values, one of them is NA, could be categorical, otherwise can drop this
index #16 - desc: might contain useful information, but not without contexts -> drop
index #17 & #18: #18 is a longer version of #17
index #19 & #20: instead of using zipcode, addr_state will be used for ease of viewing
index #23: seems like a history of borrowing, can be combined with #13 for analysis
index #31: 2 possible values, should be categorical, but only 1 exists in the dataset
index #34 & #35: seems similar, can check then drop either one
index #44 - #53: missing categorical values, represented by NA

Normally, the NA values here should be double checked with the source of data before assuming/changing for analysis. Here, that option is not on the table, so those with too much NA will be ommitted.
